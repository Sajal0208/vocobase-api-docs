# Vobiz Setup

This guide walks you through connecting your Vobiz account to Vocobase for outbound calling.

<Note>
  Vobiz credentials can be configured either via the Partner API (`PUT /api/v2/config/telephony/vobiz`) or through the Vocobase dashboard's **Telephony Connections** panel — both write to the same underlying record. Once Vobiz is configured, start calls programmatically using `POST /calls/start` with `"provider": "vobiz"`.
</Note>

## Prerequisites

- A [Vobiz account](https://www.vobiz.ai/) with a verified Auth ID and Auth Token
- At least one Vobiz phone number provisioned for outbound voice
- An approved Vocobase partner account with `"vobiz"` present in `allowed_telephony_providers` (check `GET /config`)
- Access to the Vocobase dashboard at [app.vocobase.com](https://app.vocobase.com)

<Tip>
  If `"vobiz"` is not yet listed in your partner config's `allowed_telephony_providers`, contact your Vocobase account manager to enable it.
</Tip>

## Step 1: Get your Vobiz credentials

<Steps>
  <Step title="Log in to the Vobiz dashboard">
    Go to [vobiz.ai](https://www.vobiz.ai/) and sign in to your account.
  </Step>

  <Step title="Copy your Auth ID and Auth Token">
    From your account settings, copy your **Auth ID** and **Auth Token**.

    - **Auth ID** — identifier in the form `MA_XXXXXXXX`
    - **Auth Token** — alphanumeric string

    <Warning>
      Your Auth Token grants full access to your Vobiz account. Treat it like a password.
    </Warning>
  </Step>

  <Step title="Identify the phone number you will call from">
    From your Vobiz account, copy the number you plan to use as the caller ID.

    The number must be in E.164 format (e.g., `+918011223344`) and must be enabled for outbound voice.
  </Step>
</Steps>

## Step 2: Configure Vobiz in Vocobase

You can either send the credentials via the Partner API, or set them up through the dashboard.

### Option A — Partner API (programmatic)

<CodeGroup>
```bash cURL
curl -X PUT https://api.vocobase.com/api/v2/config/telephony/vobiz \
  -H "Authorization: Bearer rg_live_abc123def456ghi789jkl012" \
  -H "Content-Type: application/json" \
  -d '{
    "auth_id": "MA_XXXXXXXX",
    "auth_token": "your_vobiz_auth_token",
    "from_number": "+918011223344"
  }'
```

```javascript JavaScript
const response = await fetch(
  "https://api.vocobase.com/api/v2/config/telephony/vobiz",
  {
    method: "PUT",
    headers: {
      "Authorization": "Bearer rg_live_abc123def456ghi789jkl012",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      auth_id: "MA_XXXXXXXX",
      auth_token: "your_vobiz_auth_token",
      from_number: "+918011223344"
    })
  }
);
const result = await response.json();
console.log(result);
```

```python Python
import requests

response = requests.put(
    "https://api.vocobase.com/api/v2/config/telephony/vobiz",
    headers={
        "Authorization": "Bearer rg_live_abc123def456ghi789jkl012",
        "Content-Type": "application/json"
    },
    json={
        "auth_id": "MA_XXXXXXXX",
        "auth_token": "your_vobiz_auth_token",
        "from_number": "+918011223344"
    }
)
print(response.json())
```
</CodeGroup>

A successful response confirms credentials were stored:

```json
{
  "success": true,
  "data": {
    "auth_id": "MA_X****XXXX",
    "from_number": "+918011223344",
    "message": "Vobiz credentials updated successfully."
  }
}
```

<Tip>
  Your Auth Token is encrypted at rest and never returned in API responses. The Auth ID is masked.
</Tip>

### Option B — Dashboard

1. Open [app.vocobase.com](https://app.vocobase.com) and sign in.
2. Go to **Settings > Telephony Connections**.
3. Click **Add Connection** and pick **Vobiz** from the provider list.
4. Paste your **Auth ID**, **Auth Token**, and **From Number**, then give the connection a name — for example, `Vobiz — Production`.
5. Click **Verify**. Vocobase calls Vobiz with your credentials to confirm they are valid; on success the connection is stored encrypted at rest.

Both paths write to the same underlying `TelephonyConnection` record — switching between API and dashboard later is fine.

## Step 3: Confirm the partner sees Vobiz as configured

Fetch your partner configuration — the response should now include Vobiz in `allowed_telephony_providers`:

<CodeGroup>
```bash cURL
curl -X GET https://api.vocobase.com/api/v2/config \
  -H "Authorization: Bearer rg_live_abc123def456ghi789jkl012"
```
</CodeGroup>

```json
{
  "success": true,
  "data": {
    "name": "acme-corp",
    "allowed_telephony_providers": ["twilio", "vobiz"],
    "telephony": {
      "twilio": { "configured": true, "account_sid": "conf****ured", "from_number": "+14155551111" },
      "mcube": { "configured": false, "exe_number": null },
      "exotel": { "configured": false, "account_sid": null, "subdomain": null, "caller_id": null, "applet_id": null },
      "plivo": { "configured": false, "auth_id": null, "from_number": null },
      "vobiz": { "configured": true, "auth_id": "conf****ured", "from_number": "+918011223344" }
    },
    "...": "..."
  }

```

`telephony.vobiz.configured: true` confirms the credentials landed and a default from-number is set. Presence in `allowed_telephony_providers` is what gates your ability to start a Vobiz call — if Vobiz isn't in that list, contact your Vocobase account manager to enable it.

## Step 4: Make a test call

Start an outbound call with `"provider": "vobiz"`:

<CodeGroup>
```bash cURL
curl -X POST https://api.vocobase.com/api/v2/calls/start \
  -H "Authorization: Bearer rg_live_abc123def456ghi789jkl012" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "to_number": "+919876543210",
    "provider": "vobiz"
  }'
```

Under the hood, Vobiz uses a self-hosted **Vobiz Media Streams** path: Vocobase asks Vobiz to dial out via the Vobiz REST API (`https://api.vobiz.ai/api/v1/Account/{authId}/Call/`), and Vobiz bridges the call into a bidirectional WebSocket served directly by the Vocobase bot at `/ws/vobiz`, carrying μ-law 8 kHz audio in both directions. This differs from the Plivo path (which is routed via Pipecat Cloud) — the bot terminates the media stream itself for Vobiz. Latency and audio quality are comparable to the other providers; partners do not need to do anything different to configure it.

## Troubleshooting

### "Vobiz not configured" or 403 on `/calls/start`

- Confirm the connection status is **Verified** in the dashboard (re-run verify if it shows as invalid).
- Confirm `"vobiz"` is in `allowed_telephony_providers` from `GET /config`. If it is not, contact your Vocobase account manager.

### 401 from Vobiz during verification

- Auth ID or Auth Token is wrong. Re-copy them from your Vobiz account settings.
- If you recently rotated the Auth Token in Vobiz, update the connection in the Vocobase dashboard or re-`PUT /config/telephony/vobiz`.

### Calls ring but drop immediately

- Confirm the destination number is reachable from your Vobiz account. Some Vobiz accounts have geographic restrictions that require explicit enablement.
- Check the Vobiz call logs for the exact reason the call was dropped (e.g., "Invalid from number", "No route to destination").

### Caller ID shows the wrong number or "Unknown"

- The `from_number` saved on your Vocobase Vobiz connection must match a number that Vobiz has provisioned for your account, in E.164 format. Update it via `PUT /config/telephony/vobiz` or the dashboard if it does not.

## Updating credentials

Rotate the Auth Token in Vobiz, then update Vocobase by re-sending `PUT /api/v2/config/telephony/vobiz` with the new token, or by editing the connection in the Vocobase dashboard. Re-run **Verify** to confirm Vocobase can still reach Vobiz with the rotated credentials.
